## Arr stack

This stack runs the media automation services used for downloading, organizing, and serving a local media library.

Services included:

| Service | URL | Purpose |
| --- | --- | --- |
| Prowlarr | `http://<server-ip>:9696` | Indexer manager for the Arr apps |
| Sonarr | `http://<server-ip>:8989` | TV show automation |
| Radarr | `http://<server-ip>:7878` | Movie automation |
| Lidarr | `http://<server-ip>:8686` | Music automation |
| qBittorrent | `http://<server-ip>:8080` | Torrent downloader |
| Jellyfin | `http://<server-ip>:8096` | Media server |
| Homarr | `http://<server-ip>:7575` | Dashboard |

### Prerequisites

- Docker and Docker Compose plugin installed.
- A fixed location for the stack data, downloads, and media library.
- NVIDIA Container Toolkit installed if you want Jellyfin hardware acceleration with the GPU settings in `compose.yml`.

### Step 1 - Create the `.env` file

Create `sisyphus/docker-accel-node/arr-stack/.env`:

```bash
PUID=1000
PGID=1000
TZ=Asia/Kolkata
ARRPATH=/home/docker/arr-stack/
```

Change `PUID` and `PGID` to the user that should own the files created by the containers:

```bash
id
```

Keep the trailing slash in `ARRPATH`. The compose file joins paths like `${ARRPATH}Downloads`, so without the trailing slash the paths will be wrong.

### Step 2 - Create the data directories

Create the directories used by the compose file:

```bash
sudo mkdir -p /home/docker/arr-stack/{Downloads,Prowlarr/config,Prowlarr/backup,Sonarr/config,Sonarr/backup,Sonarr/tvshows,Radarr/config,Radarr/backup,Radarr/movies,Lidarr/config,Lidarr/music,Homarr/configs,Homarr/icons,Homarr/data,Jellyfin/config,qbittorrent/config}
sudo chown -R 1000:1000 /home/docker/arr-stack
```

If you changed `PUID` and `PGID` in `.env`, use those same values in the `chown` command.

### Step 3 - Start the stack

From this directory:

```bash
docker compose up -d
```

Check the containers:

```bash
docker compose ps
```

View logs for a service:

```bash
docker compose logs -f sonarr
```

### Step 4 - Configure qBittorrent

Open qBittorrent:

```text
http://<server-ip>:8080
```

Set the default save path to:

```text
/downloads
```

The first login credentials are generated by the linuxserver qBittorrent image. Check them with:

```bash
docker compose logs qbittorrent
```

After logging in, change the qBittorrent password.

### Step 5 - Configure Prowlarr

Open Prowlarr:

```text
http://<server-ip>:9696
```

Add your indexers first, then connect Prowlarr to the Arr apps:

| App | Host inside Docker | Port |
| --- | --- | --- |
| Sonarr | `sonarr` | `8989` |
| Radarr | `radarr` | `7878` |
| Lidarr | `lidarr` | `8686` |

Get each app API key from:

```text
Settings -> General -> Security -> API Key
```

### Step 6 - Configure Sonarr, Radarr, and Lidarr

Use these root folders:

| App | Root folder |
| --- | --- |
| Sonarr | `/data/tvshows` |
| Radarr | `/data/movies` |
| Lidarr | `/data/musicfolder` |

Add qBittorrent as the download client in each Arr app:

| Setting | Value |
| --- | --- |
| Host | `qbittorrent` |
| Port | `8080` |
| Category for Sonarr | `tv` |
| Category for Radarr | `movies` |
| Category for Lidarr | `music` |

The download path seen by the Arr apps and qBittorrent is the same container path:

```text
/downloads
```

### Step 7 - Configure Jellyfin

Open Jellyfin:

```text
http://<server-ip>:8096
```

Add libraries using these paths:

| Library type | Path |
| --- | --- |
| Movies | `/data/Movies` |
| TV Shows | `/data/TVShows` |
| Music | `/data/Music` |

If Jellyfin does not start because the server has no NVIDIA GPU or NVIDIA Container Toolkit, remove the Jellyfin `environment` and `deploy.resources.reservations.devices` GPU sections from `compose.yml`, then run:

```bash
docker compose up -d
```

### Step 8 - Configure Homarr

Open Homarr:

```text
http://<server-ip>:7575
```

Add links for the services:

```text
Prowlarr: http://<server-ip>:9696
Sonarr: http://<server-ip>:8989
Radarr: http://<server-ip>:7878
Lidarr: http://<server-ip>:8686
qBittorrent: http://<server-ip>:8080
Jellyfin: http://<server-ip>:8096
```

### Updating the stack

```bash
docker compose pull
docker compose up -d
```

Remove old unused images:

```bash
docker image prune -a
```

### Backup

Stop the stack before making a full backup:

```bash
docker compose down
sudo tar -czf arr-stack-backup.tar.gz /home/docker/arr-stack
docker compose up -d
```

At minimum, back up each service config directory:

```text
Prowlarr/config
Sonarr/config
Radarr/config
Lidarr/config
Jellyfin/config
qbittorrent/config
Homarr/
```

docker compose restart
